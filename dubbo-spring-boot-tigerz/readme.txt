===工程关系===
tigerz是父工程，下面有子工程模块api，gm，aus
api  提供基本服务接口
gm   提供IP地址查询跳转，账号系统
aus  是澳大利亚房产服务

tigerz的父工程是parent
    作用：主要是文件夹作用，同时还有spring-web已经我们自己需要的库
    这一层也可以不要。主要是为了跟框架兼容隔离，未来框架有更新，直接拷贝过来就行
    不需要做任何修改

以上都是我们自己东西，修改主要上上面修改

以下都是框架的东西，我们没有做任何修改。如果框架更新则直接覆盖即可
    
parent的父工程是project
    作用：加入了基础的依赖，比如zookeeper，spring-boot-dependencies，dubbo
project是最上级父工程
    作用：定义了整个工程基本信息，如果开发者等，整体版本
    其实主要还是文件夹的作用，所有的模块都在这里注册
